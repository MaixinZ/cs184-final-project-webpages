<div className="page-shell">
  <section className="hero-note">
    <p className="note-kicker">overview</p>
    <h1>Real-Time Multi-Style Rendering and Interactive Style Composition in a Shared 3D Scene</h1>
    <div className="hero-lead">
      We propose a real-time graphics project that explores how a single 3D
      scene can be rendered in multiple distinct visual styles while preserving
      the perception of one coherent geometric world. Rather than treating
      different styles as separate environments, we aim to maintain shared
      geometry, spatial layout, object identity, and lighting logic while
      changing the visual language through shading, material response,
      post-processing, and limited geometric modulation. A central extension of
      this project is interactive style composition: in addition to switching
      the style of the entire scene, users can define a screen-space boundary
      such as a line or a circle to split the image into regions, with each
      region rendered under a different style.
    </div>
    <div className="hero-meta">
      <span>real-time rendering</span>
      <span>multi-style composition</span>
      <span>shared 3D scene</span>
      <span>interactive visualization</span>
    </div>
  </section>

  <section>
    <article className="paper-card">
      <span className="paper-label">Project framing</span>
      <h2>What this project is trying to conquer?</h2>
      <div className="copy-block">
        How can one shared 3D scene be rendered in multiple visual styles,
        globally or locally, while still feeling like the same physical space?
      </div>
      <ul className="metric-strip">
        <li>
          <strong>Global</strong>
          one scene, one style pass, fully transformed
        </li>
        <li>
          <strong>Local</strong>
          multiple styles coexisting in one rendered image
        </li>
        <li>
          <strong>Constraint</strong>
          consistency across geometry, lighting, and motion
        </li>
      </ul>
      <br/>
      <span className="paper-label">Goal 1: Global style transformation</span>
      <h3>Render the entire scene under different styles</h3>
      <div className="copy-block">
        This direction studies full-scene style transformation at render time:
        painterly, toon-like, sketch-like, or other shading languages applied
        consistently to the whole view. It also gives a clean space for
        discussing or benchmarking different shading styles and asking which
        ones preserve scene readability best.
      </div>
      <ul className="stack-list">
        <li>
          <strong>Discussion / benchmarking</strong>
          Compare shading models, edge treatments, post effects, and material remappings across several global styles.
        </li>
        <li>
          <strong>Main difficulty</strong>
          After applying a global style, the result still has to stay visually consistent and must not produce locally weird patches or broken style cues.
        </li>
      </ul>
      <br/>
      <span className="paper-label">Goal 2: Local interactive style partitioning</span>
      <h3>Allow multiple styles to coexist within one rendered view</h3>
      <div className="copy-block">
        This direction focuses on region-based composition: users can drag a
        split line, mask, or circle so two or more styles appear inside the
        same rendered image. The target feeling is closer to a dual-world
        presentation, similar to the kind of side-by-side coexistence seen in
        <em> Split Fiction </em>, but kept inside one coherent camera view.
      </div>
      <ul className="stack-list">
        <li>
          <strong>Interaction goal</strong>
          Let style boundaries move live without breaking the sense of shared geometry.
        </li>
        <li>
          <strong>Hard part</strong>
          The boundary should feel deliberate, not like two unrelated renders stitched together with mismatched lighting, silhouettes, or depth cues.
        </li>
      </ul>
    </article>
  </section>
</div>
